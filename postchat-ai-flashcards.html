PostChat AI – Flashcards

Turn long AI conversations or pasted text into structured flashcards.

Overview

PostChat AI is an iOS app for converting long AI conversations or pasted text into flashcards. It’s built to take messy, exploratory content (chat transcripts, notes, long threads) and turn it into a study-ready deck, without requiring manual rewriting.

This project also served as a higher-complexity “app factory” validation on top of my shared platform: it stresses async workloads, multi-step generation, cost controls, and a real end-to-end workflow (input → processing → deck → study).

Built with

  • Idea: The concept came from repeatedly using ChatGPT/Codex for long technical threads and then wanting a way to actually study the outcomes.
  • Graphic Design: Designed the screens and flows using Figma Make. Iterated and generated UI variants through the Figma MCP Server to move faster on layout, spacing, and flow changes.
  • Architecture Design: Built on the shared Learn & Be Curious iOS template (sidebar shell, auth surfaces, legal links, shared navigation patterns). Uses the shared platform backend for identity and multi-tenant separation. Introduces an app-specific data domain for conversations/summaries/decks/cards. Flashcard generation runs asynchronously so the UI isn’t blocked by model calls.
  • Requirements: Paste mode as the default Home flow. Chat mode as Pro-gated. Framing prompt required before generation. Mandatory pipeline: chunking → summarization → semantic summary → flashcard generation. Deck persistence + study mode (flip/swipe/expand) with simple UX.
  • Development: Vibe-coded implementation using OpenAI Codex for the iOS client and backend pipeline scaffolding. Integrated server-side OpenAI calls for summarization + flashcard generation with bounded inputs. Built the deck + card study UI on top of the shared template shell.
  • Testing: Manual end-to-end testing of paste and chat flows: ingestion, framing, async generation, deck creation, and study mode interactions. Sanity checks around long inputs (chunking behavior) and stable generation completion.
  • Deployment: Built and shipped via the shared CI workflow (GitHub Actions) using manifest + overlay patterns. Backend deployed to shared environments; iOS builds produced as distributable artifacts (TestFlight/App Store links TBD).

AI inside

Yes. Uses OpenAI models server-side for summarization and flashcard generation. Also uses AI tooling during development (Codex) and design iteration (Figma Make + Figma MCP Server).

Tech Stack

  • iOS: Swift, SwiftUI (UIKit where needed)
  • Backend: Serverless platform backend (API Gateway, Lambda)
  • Database: Postgres (app-specific domain)
  • Async: Job-based pipeline (queue/workers)
  • AI: OpenAI APIs (server-side) + OpenAI Codex (development)
  • Design: Figma Make + Figma MCP Server
  • Build/automation: GitHub Actions reusable workflows
  • Shared platform: Template shell + auth/profile infrastructure
