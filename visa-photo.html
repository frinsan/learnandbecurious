Visa Photo (Passport Photo Maker)

A lightweight iOS utility for creating consulate-friendly passport and visa photos in preset or custom sizes.

Overview

Visa Photo is a lightweight iOS app for creating passport and visa photos in common consulate-friendly sizes. It was the first “real” branded app built on top of my shared iOS template and backend, used to validate the end-to-end app factory workflow (manifest + overlay + automated builds) with a simple, deterministic product.

The core user flow is intentionally minimal: capture or upload a photo, choose a preset (visa/passport) or enter a custom size, align the face using a cropping guide with zoom/drag controls, and export the final image at the target dimensions.

Built with

  • Idea: I needed a straightforward passport/visa photo utility for my wife’s visa application, but most App Store options were either upsell-heavy or priced like a one-off purchase ($10+). Build a very small app as the first customer of the template to prove the app-factory concept: brand repo + manifest + overlay code, without forking the template. Keep the UX simple and predictable; prioritize “works every time” exports over complex editing features.
  • Graphic Design: No dedicated design toolchain. The app relies on template theming plus a small set of brand assets (app icon, guide overlays). The UI is designed to feel like a focused utility: a single primary call-to-action and a linear flow.
  • Architecture Design: Built as a brand overlay on top of the shared SwiftUI template: brand-specific Home replaces the template Home experience (PassportHomeView + PhotoFlowView). Photo size presets live in a small JSON file (presets_library.json) loaded at runtime. Uses the shared template shell (sidebar, legal links, optional account/login surfaces) so future apps can share the same infrastructure.
  • Requirements: Support a small set of visa/passport presets (mm + DPI) plus a custom size option. Provide an on-screen guide silhouette for 1:1 and rectangular formats to help alignment. Export consistently-sized images suitable for digital submission. Keep the app fast, offline-friendly, and minimal.
  • Development: Implemented a guided, step-based flow: UploadPhoto (camera or photo library selection), ChooseFormat (select a preset or custom dimensions), Edit (pinch-to-zoom and drag-to-align inside a target frame with an overlay guide), and Export (generate output image data and save to the Photos library, with optional share sheet UX available in the flow). Packaged as a brand repo with a manifest (bundle id, URL scheme, theme, feature flags) and an overlay that replaces only the pieces that are app-specific.
  • Testing: Manual testing focused on the flow end-to-end: import/capture, preset selection, crop/zoom/drag behavior, and successful export/save. Sanity checks for common preset sizes (e.g., 2x2 inch style formats) and consistent pixel dimensions at 300 DPI.
  • Deployment: Designed to be built via the template’s reusable CI workflow: apply the manifest, apply overlay code/assets, archive, and produce a distributable artifact for TestFlight/App Store Connect. Backend integration (auth/profile endpoints) comes from the shared platform; the photo workflow itself is local to the device.

AI inside

None. The app is intentionally non-AI and focused on deterministic photo sizing/cropping.

Tech Stack

  • iOS: Swift, SwiftUI
  • Image pipeline: UIKit (UIImage), ImageIO, UniformTypeIdentifiers
  • Device integration: Photos framework (save/export), camera/photo picker UI
  • Config: JSON preset library + per-brand manifest (app.json)
  • Build/automation: GitHub Actions reusable workflows + template build scripts
  • Shared platform (template-provided): Cognito Hosted UI auth + platform backend (API Gateway, Lambda, DynamoDB) for account/profile flows
