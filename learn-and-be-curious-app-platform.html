Learn and Be Curious: App Platform (iOS App Factory)

A shared backend, SwiftUI template, and automation system that makes shipping branded iOS apps fast and repeatable.

Overview

Learn and Be Curious is a “platform app factory” designed to make it easy to build, brand, and ship many iOS apps without starting from scratch each time. Instead of creating a new codebase per app, the system centralizes everything that should be shared (auth, networking, backend services, CI/CD patterns, observability, security defaults) and pushes everything that should vary per app into a small brand repo (manifest + assets + optional overlay code).

At a high level, the platform has three moving parts:

  1. Platform backend (platform-backend/): a multi-tenant AWS serverless backend with staging and prod environments, Cognito auth (Apple/Google Hosted UI + optional native email), a versioned API (/v1/...), and DynamoDB persistence keyed by appId.
  2. iOS template app (ios-app-template/): a working SwiftUI starter app with a sidebar shell, manifest-driven configuration, auth/session handling, and a typed API client.
  3. Brand repos (e.g., app-sample/): a lightweight repository per app that supplies app.json (bundle id, display name, theme, API/auth config, feature flags, legal links, versions) plus assets and optional Overlay/TemplateApp/... Swift overrides.

The “factory” mechanism turns a brand repo into a buildable iOS project by creating a scratch copy of the template and applying the brand’s manifest + overlay automatically (bundle ID, Info.plist, URL scheme, app icon/assets, versions). This keeps the template repo clean and makes iteration fast while supporting CI workflows that can produce signed archives for TestFlight/App Store distribution.

Built with

  • Idea: The goal was to standardize and accelerate shipping many small, branded apps by making “new app creation” mostly configuration and assets rather than new code.
  • Collaboration: OpenAI ChatGPT and Codex were used as iterative collaborators to explore architecture options, define onboarding workflows, and implement end-to-end scaffolding quickly.
  • Graphic Design: No dedicated design toolchain. Branding is driven by per-app assets (icons/splash/colors) and a manifest-controlled theme (primary/accent/appearance).
  • UI System: Template-first: a consistent sidebar shell and a neutral baseline that can be overridden per brand via overlays.
  • Architecture Design: Split architecture: one shared backend, one shared iOS template, many per-brand repos. Multi-tenant backend with data partitioning and authorization keyed by appId.
  • Environments: Two environments (staging/prod) to support QA/TestFlight builds before production promotion.
  • Automation: Automation-first scripts and reusable workflows compose the final branded app at build time rather than forking the template codebase per brand.
  • Requirements: Ship a working baseline app (sidebar shell + login + blank home), manifest-driven configuration, Cognito Hosted UI auth, and a repeatable new-brand launch process.
  • Development: Monorepo workspace for backend, iOS template, and example brand repos. build_brand.sh creates scratch copies; apply_manifest.sh validates JSON and rewrites bundle IDs, Info.plist display name, URL schemes, versions, and assets.
  • Testing: Template includes TemplateAppTests and can be exercised via xcodebuild test; primary validation is end-to-end smoke testing in staging before production.
  • Deployment: Backend deploys via AWS CDK with separate staging/prod stacks; iOS builds produced locally or in GitHub Actions workflows for TestFlight/App Store distribution.

AI inside

The iOS template includes a feature-flagged “AI Playground” screen used to validate AI plumbing. When available, it uses Apple’s on-device Foundation Models integration (SystemLanguageModel via FoundationModels) to rewrite text (clearer/more formal/shorter). When unavailable (simulators or unsupported devices), it falls back to deterministic local transforms so the UI remains testable and predictable.

Tech Stack

  • iOS app template: Swift, SwiftUI, XCTest
  • Auth (mobile): Cognito Hosted UI (Sign in with Apple + Google), custom URL scheme redirect handling
  • Backend IaC: AWS CDK (TypeScript)
  • Backend services: Amazon API Gateway (HTTP API), AWS Lambda, Amazon DynamoDB
  • Edge/static assets: Amazon S3, Amazon CloudFront
  • Observability: Amazon CloudWatch logs and alarms
  • CI/CD: GitHub Actions (reusable workflows; xcodebuild test/archive; artifact upload)
  • Build tooling: Bash scripts, jq (manifest validation), rsync (scratch copy), PlistBuddy + perl (project/Info.plist rewrites)
